:::info
你需要 Java 17 或更改的版本以运行 Geyser.
:::

:::caution
Geyser-Standalone 不是一个插件或 mod! 它是一个独立的程序用来连接 Java 服务器.
:::

#### 安装 & 配置 \{#general-setup--configuration}

1. [点击下载合适的 Geyser 版本](/download).
2. 创建一个新文件夹, 将 Geyser 放入其中.
3. 运行:
   - **GUI** (Recommended):   
   双击 .jar 文件.
   或创建一个其中脚本以启动(详见[这](/wiki/geyser/creating-a-startup-script/))

4. 打开 Geyser 配置文件夹内的 `config.yml`, 你将会看到如下内容:

   ```yaml
    bedrock:
      # 允许访问的 IP 地址
      # 通常来说, 你不应该取消注释, 除非你要限制只有来自某个 IP 地址的访问被允许.
      #address: 0.0.0.0

      # 基岩版 服务器端口.
      port: 19132
   ```
   以上内容是配置模拟的基岩版服务器的信息.
   然后, 你需要配置你需要连接到的 Java 服务器信息.
   ```yaml
   remote:
     # Java 服务器的 IP 地址(一般用于独立版 Geyser).
     # 对于独立版本来说, "auto" 代表着 127.0.0.1 .
     # 对于其他版本来说, 最好保持为 "auto", 以便 Geyser 自动配置地址, 端口, 客户端登录方式 和 一些 Floodgate 配置.
     address: auto
     # Java 服务器端口.
     # 插件版本可设置为 "auto" 自动选择端口.
     port: 25565
     # 登录方式. 可为 "offline", "online", 或者 "floodgate" (详见 https://github.com/GeyserMC/Geyser/wiki/Floodgate).
     # 建议将 `address` 项保持为 "auto" 以便于自动进行 Floodgate 配置.
     # 如果安装了 Floodgate 并且 `address:` 设置项设为 "auto", 那么本设置项将自动改为 "floodgate".
     auth-type: online
   ```
   现在你可以配置你所想要连接的 Java 服务器了.
   如果你所连接的服务器上安装了 Floodgate, 那么推荐你将 `auth-type` 更改为 "floodgate"([详见](/wiki/floodgate/setup?platform=standalone).
   
5. 连接到你的服务器:

   **本地连接**
   在同台主机上, 你可以使用 `localhost`, 或者是 `127.0.0.1` 作为连接地址.
   你可能遇到环回限制(Loopback restrictions), 详见[此处修复](/wiki/geyser/fixing-unable-to-connect-to-world#using-geyser-on-the-same-computer).
   在同一子网下的其他设备可以使用你的本地 IPv4 作为连接地址.
   或者你可以通过基岩版客户端中的 好友 选项卡加入服务器.

   **远程连接**
   就像让你的 Java 服务器可以被连接一样进行操作.
   不同的是, 你必须保证 Geyser 端口必须使用 UDP 协议.

6. 使用 `geyser connectiontest <ip> <port>` 命令验证是否能够连接.

#### 在安卓上运行 Geyser 独立版 \{#running-geyser-standalone-on-android}

:::caution
安卓上的 Termux 等应用可以用来运行 Geyser, 但是它完全取决于你的设备如何, 请自行承担风险.
:::

##### Termux (Android) \{#termux-android}
1. 下载并安装 [Termux](https://termux.com/)
2. 运行 `pkg install openjdk-17`
3. 运行 `wget https://download.geysermc.org/v2/projects/geyser/versions/latest/builds/latest/downloads/standalone`
4. 运行 `java -jar Geyser-Standalone.jar`

或者说

我们有一个用于 Termux 的自动安装脚本, 但它并不适用于所有用户. 如果上面的操作不起作用, 请尝试使用此方法. 运行此命令以开始下载并安装:
```
curl https://gist.githubusercontent.com/rtm516/e3e07d6595ee41e05a38b03c0f4d7a80/raw/install.sh | bash
```
